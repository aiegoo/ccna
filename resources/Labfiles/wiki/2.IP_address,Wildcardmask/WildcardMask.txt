                                    ### WildcardMask

-EIGRP , OSPF Routing protocol의 네트워크의 범위를 지정할때 사용된다.

-ACL등과 같은 Filtering기능을 사용하여 Filtering을 실시할 네트워크의 범위를 지정할때 사용된다.


                                ## SubnetMask와 WildcardMask의 차이점

                # SubnetMask
-32bit로 구성되며 1과 0의 연속적인 형식만 가능하다.
 .1 = Network (공통비트)
 .0 = Host (비 공통비트)



                # WildcardMask
-32bit로 구성되며 1과 0의 불연속한 형식이 가능하다.
 .0 = 공통비트
 .1 = 비 공통비트

-192.168.1.0/24
  :  192.168.1.0  0.0.0.255	(192.168.1.0 ~ 192.168.1.255)


-172.16.0.0/16
  :  172.16.0.0  0.0.255.255	(172.16.0.0 ~ 172.16.255.255)


-10.0.0.0/8
  :  10.0.0.0  0.255.255.255	(10.0.0.0 ~ 10.255.255.255)


-192.168.1.0/27
  :  192.168.1.0  0.0.0.31	(192.168.1.0 ~ 192.168.1.31)


-192.168.1.64/26
  :  192.168.1.64  0.0.0.63	(192.168.1.64 ~ 192.168.1.127)


-192.168.1.1  0.0.0.0 = Host
